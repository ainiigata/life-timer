# 人生タイマー v2 第1弾 設計ドキュメント

日付: 2026-07-06
ステータス: 承認待ち(要件・方針はユーザー確認済み)
前提: v1(残り寿命カウントダウンPWA)は公開済み。本書はその機能追加の第1弾。

## 0. 全体方針(段階リリース)

行動を促す機能群を2段階で出す。本書は**第1弾**のみを対象とする。

- **第1弾(本書)**: 新しい個人データをほぼ増やさず、既存の残り時間データだけで作れる高インパクト機能 — 格言バナー / 新タブ「見つめる」/ 今日の宣言
- **第2弾(別スペック)**: 家族の続柄→抱っこ・同居できる残り日数、優先順位(ふりかえり型)+夢ピン留め、Life in Weeks格子

## 1. 追加する機能

### ① 格言バナー(「わたし」画面の上部)
- 偉人の格言(人物名+言葉)を起動時にランダム表示。タップで次へ。重複を避けて巡回する。
- 収録数は25〜30。核となる5つ(下記)は必ず含める。時間・行動・死・今を生きることをテーマにする。
- **正確性**: 文言と人物は、収録前に一次情報・信頼できる出典で裏取りする(生命表と同じ規律)。出典が曖昧・諸説あるものは避けるか、意訳と分かる形にする。誤帰属は載せない。
- 核の5つ(ユーザー指定):
  - スティーブ・ジョブズ:「もし今日が人生最後の日だとしたら、今日やろうとしていることを本当にやりたいか?」
  - 中島敦:「人生は何事もなさぬにはあまりにも長いが、何事かをなすにはあまりにも短い」
  - ガンジー:「明日死ぬかのように生きよ。永遠に生きるかのように学べ」
  - セネカ:「致命的なのは、時間が足りないことではなく、多くの時間を無駄にしていることだ」
  - 吉田松陰:「今日できることを明日に延ばすな。今日という日は二度と来ない」

### ② 新タブ「見つめる」(下タブに4つ目を追加)
タブ順: わたし / **見つめる** / 家族 / やりたいこと。残り時間から自動計算するカードを縦に並べる。本人(誕生日・性別)未設定時は「まず『わたし』で誕生日を設定してください」と案内。

カード一覧(すべて算出値):
1. **残り秒数**: 「約 12億3456万秒」。1秒ごとに減る。
2. **残りの呼吸数**: 残り秒 × 16/60。「約 ○億回」。
3. **残りの心臓の鼓動**: 残り秒 × 70/60。「約 ○億回」。
4. **残り時間の内訳**: 睡眠(1日8時間=1/3)で消える時間を年換算。「眠っている時間だけで、あと約○年」。加えて自由に使える時間(起きている時間)も。
5. **あと何回**: 桜(年1)/ 誕生日(年1)/ 年末(年1)/ 満月(年約12.37)を死亡推定年まで。「あと桜を○回」。
6. **明日死ぬ確率**(前向きトーン): 生命表の死亡率(qx)から日次確率を算出。「明日を無事に迎えられる確率 99.99%」+「宝くじ1等(約2000万分の1)より、明日が来ない可能性のほうが約○倍高い」+「だから、今日を大切に」。
7. **親に育ててもらった日数**: 本人の生きた日数。「あなたが生きてきた ○○日、親が見守ってくれた日々」。
8. **今日の残り**: 起きている時間の残り。就寝時刻は23:00固定(将来設定化)。「今日、起きていられるのはあと○時間○分」。

タブ下部に注記(常時): 「呼吸・鼓動・確率は統計からのおおよその目安です。医療情報ではありません。」

### ③ 今日の宣言(「わたし」画面、カウントダウンの下)
- 当日分が未設定: 「今日やることを1つ」入力欄+「決める」ボタン。
- 設定済み: 宣言文+チェックボタン。チェックで達成(小さなお祝い演出、既存 celebrate を流用)。
- 日付が変わったら当日分はリセット(新規入力に戻る)。前日の宣言は保持しない(第1弾では履歴なし)。

## 2. データモデル変更

localStorage キー `life-timer-v1` は据え置き(バージョンは1のまま。後方互換で追加)。

```json
{
  "version": 1,
  "self": { ... },       // 変更なし
  "family": [ ... ],     // 変更なし
  "wishes": [ ... ],     // 変更なし
  "today": { "date": "2026-07-06", "text": "履歴書を書く", "done": false }  // 追加・任意(null可)
}
```

- `today` は任意。古いデータ(today なし)もそのまま読める。`validate` は today を「null または {date:非空文字列, text:非空文字列, done:boolean}」として許容する。
- `emptyData()` は `today: null` を含める。
- 格言・生命表・各種係数は静的データでユーザー保存対象外。

## 3. 追加する静的データ・計算

### life-table.js に死亡率(qx)を追加
- 令和5年簡易生命表の「死亡率」列(nqx: 年齢xの人が1年以内に死ぬ確率)を男女別・5歳刻みで同梱(平均余命exと同じ一次資料 life23-1x.xlsx から取得。前回ダウンロード済み)。
- `LifeTable.annualMortality(gender, age)`: exと同様に線形補間で年間死亡率を返す。

### time-calc.js に純粋関数を追加(テストファースト対象)
- `remainingSeconds(now, death)` → 残り秒(整数、期限切れは0)
- `countByRatePerMinute(remainingSec, perMinute)` → floor(残り秒 × perMinute / 60)。呼吸(16)・鼓動(70)に使う
- `occurrencesUntil(now, death, perYear)` → floor(残り年 × perYear)。あと何回(桜=1・誕生日=1・年末=1・満月=12.37)に使う
- `daysLived(birthDate, now)` → 生きた日数(整数)
- `dailyDeathProbability(qxAnnual)` → 1 − (1 − qx)^(1/365)。日次死亡確率
- `awakeRemainingToday(now, bedHour)` → 就寝時刻までの {hours, minutes}。過ぎていれば {0,0}

### insight.js(新規モジュール): 表示用ビューモデルを組み立てる純粋関数
- `Insight.build(self, deathDate, now)` → カードのビューモデル配列 `[{ id, label, value, sub }]` を返す。
  - LifeTable / TimeCalc を使い、数値の算出と整形(「約○億」表記、桁区切り等)を担う。DOMには触れない。
- こうしておくと app.js は配列をDOMに描画するだけになり、計算・整形をテストで検証できる。

### quotes.js(新規モジュール)
- `Quotes.LIST` = `[{ author, text }]` の配列(裏取り済み)。
- 巡回の順序管理(シャッフル・重複回避)は app.js が持つ。

## 4. モジュール構成とビルド

既存のデルマエ方式(マーカー注入)を踏襲。新モジュールを追加する:

- 追加ファイル: `src/quotes.js`(UMD, `Quotes`)、`src/insight.js`(UMD, `Insight`, 依存: LifeTable, TimeCalc)
- `build.js` の INJECTIONS に `QUOTES` `INSIGHT` を追加(依存順に注入: LIFE_TABLE → TIME_CALC → QUOTES → INSIGHT → STORE → APP)
- `src/index.template.html` に script タグを2つ追加(合計6本)
- `test/build.test.js` の「script開閉タグ N 組」assertion を 4→6 に更新
- app.js は state と DOM 配線に集中。見つめるカードの描画 `renderInsight()`、格言バナー、今日の宣言を追加。app.js は現状378行 → 550〜600行程度に増える見込み。単一UIコントローラーという既存パターンの範囲内として許容し、計算・整形は insight.js / time-calc.js 側に置いて肥大を抑える。

## 5. 画面・DOM

- タブバー: `data-tab="insight"` のボタンを「わたし」の次に追加。`#screen-insight` セクションを追加。4タブが375pxに収まる(各約93px)。
- 「わたし」画面: 最上部に格言バナー `#quote-banner`(author/text)、カウントダウンの下に今日の宣言 `#today-declaration`。
- 「見つめる」画面: `#insight-cards` にカードを動的生成。未設定時プレースホルダ `#insight-empty`、下部に注記。
- 毎秒更新(tick): アクティブな画面のみ再描画。見つめるが active のとき renderInsight() を毎秒呼ぶ(残り秒カードのため)。DOMは差分ではなく textContent 更新で軽量に。

## 6. XSS・入力の扱い

- 今日の宣言テキスト・格言テキストはユーザー/静的いずれも `textContent` で注入(innerHTML に生値を入れない)。既存カードと同じ規律。
- 今日の宣言は最大長(例: 60文字)で制限。空文字は登録不可。

## 7. エラー・境界

- 本人未設定: 見つめるタブは案内文のみ。格言バナーは本人未設定でも表示してよい(データ不要)。
- 期限切れ(統計を超えて存命): 残り秒=0、確率カードは「統計上の平均を超えて生きています」を優先表示。
- 日付境界: 今日の宣言は `today.date !== 今日` で当日未設定として扱う。うるう・月末は既存 time-calc の日付処理に準拠。
- 就寝時刻を過ぎた深夜: 今日の残り=0時間、「そろそろ休みましょう」等。

## 8. テスト方針(node --test, npm依存ゼロ)

- time-calc の追加関数: remainingSeconds(期限切れ0含む)、occurrencesUntil(端数切り捨て)、daysLived(うるう跨ぎ)、dailyDeathProbability(既知qxでの値域)、awakeRemainingToday(就寝前・就寝後)。
- life-table.annualMortality: 掲載年齢一致・線形補間・境界。qx値は一次資料と照合(前回同様、実装時にxlsxと突き合わせ)。
- insight.build: 本人設定あり/なし、代表ケースでラベル・整形(「約○億」桁)を検証。DOM非依存なので node で直接テスト可能。
- store: today の validate(正常・型違い・古いデータ後方互換)、往復。
- build: script タグ6本、INJECTマーカー残存なし、外部URLゼロ、`fetch(` 不使用(sw.js除く)を維持。

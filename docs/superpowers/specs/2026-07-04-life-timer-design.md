# 人生タイマー 設計ドキュメント

日付: 2026-07-04
ステータス: 承認済み(要件・設計ともユーザー確認済み)

## 1. コンセプト

残り時間を秒単位で可視化して「今やる」行動を引き出すアプリ(memento mori × 夢を叶える)。
完全ローカル・通信ゼロ・無料。誕生日等の個人情報は端末外に出ない。

- ターゲット: まず本人と家族。GitHub Pagesで一般公開もする
- トーン: 前向き・夢を叶える(明るめ配色、達成演出あり)
- 検証したい仮説: 残り時間が減り続ける画面を毎日見ると、やりたいことリストの実行率が上がる

## 2. MVP機能(3つ)

1. **残り時間タイマー(本人)** — 誕生日+性別入力 → 「◯年◯ヶ月◯日 ◯時間◯分◯秒」をリアルタイム表示+人生プログレスバー(◯%消化)
2. **家族タイマー+会える回数換算** — 家族を複数登録(名前・誕生日・性別・会う頻度)。「月1で会うなら あと◯回」を大きく表示
3. **やりたいことリスト** — 追加・達成チェック。任意で「◯歳までに」を設定すると残り年月を項目に表示。達成時に小さなお祝い演出

後回し: 人生年表、Life in Weeks格子、通知、テーマ切替、共有機能、ウィジェット

## 3. 技術方針

- **単一HTML + vanilla JS + localStorage**。npm依存ゼロ、外部API呼び出しゼロ、従量課金要素ゼロ
- デルマエ方式: `src/` 分割開発 → `node build.js` で `dist/index.html` にインライン化
- PWA: `dist/` に `index.html` / `manifest.json` / `sw.js` / アイコン。Service Workerはキャッシュファーストの最小構成で完全オフライン動作
- CSPで外部接続を遮断
- 公開: GitHub Pages(無料)
- UI実装は frontend-design スキルを使用。スマホファースト(375px基準)

## 4. データモデル

localStorage キー: `life-timer-v1`

```json
{
  "self":   { "name": "", "birthDate": "1985-06-15", "gender": "male|female", "customLifespan": 90 },
  "family": [
    { "id": "", "name": "母", "birthDate": "", "gender": "", "customLifespan": null,
      "meetFrequency": "daily|weekly|monthly|yearly-N" }
  ],
  "wishes": [
    { "id": "", "title": "", "targetAge": 45, "done": false, "createdAt": "", "doneAt": null }
  ]
}
```

- `customLifespan` は任意(「100歳まで生きるつもり」派向けの上書き)
- `meetFrequency` の `yearly-N` はNが年間回数(例: 帰省年2回 = `yearly-2`)

## 5. 余命計算仕様

- 厚労省 簡易生命表(令和5年)の年齢別平均余命を `src/life-table.js` に男女別配列で同梱(0〜105歳、出典コメント明記)
- 期待寿命 = 現在年齢の平均余命(端数年齢は線形補間で算出)+ 現在年齢
- `customLifespan` 設定時はそちらを優先
- 残り時間表示 = 期待死亡日時 − 現在時刻 を「年/ヶ月/日/時間/分/秒」に分解。`setInterval` 1秒更新
- 人生プログレスバー = 経過時間 ÷ 期待寿命全体(%)
- **会える回数** = 残り年数 × 頻度係数(daily=365, weekly=52, monthly=12, yearly-N=N)。
  本人と家族の残り時間の**短い方**を上限とする(先に尽きる側までしか会えない)
- 余命テーブル上限(105歳)を超える年齢は最小余命値で処理

## 6. 画面構成(下タブ3画面)

- **わたし**: 初回はオンボーディング(誕生日・性別入力)。以降は大きなカウントダウン+プログレスバー+前向きメッセージ。設定(誕生日修正・目標寿命・JSON書き出し/読み込み)もここから
- **家族**: 家族カード一覧(名前・残り時間簡易表示・「◯◯で会うなら あと◯回」を大きく)。追加・編集・削除
- **やりたいこと**: 未達成/達成でセクション分け。「◯歳までに」設定時は残り年月を表示。達成チェックでお祝い演出

## 7. エラー処理

- localStorage 破損時: 黙って初期化しない。「バックアップから復元 or リセット」を提示
- JSONインポート: バリデーション(スキーマ・日付妥当性)を通してから反映。不正なら理由を表示して現データ保持
- 誕生日が未来日付・150年以上前は入力時に拒否

## 8. テスト方針

`node --test`(npm依存ゼロ)でロジックをテストファースト:

- 余命計算(線形補間・customLifespan優先・105歳超)
- 残り時間の年月日時分秒分解(うるう年、誕生日当日、月末)
- 会える回数換算(全頻度、家族が先に尽きるケース)
- JSONインポートのバリデーション(正常系・破損データ・型違い)
- ビルド後のdist検証(マーカー残存なし等)はデルマエのtools/構成を参考に
